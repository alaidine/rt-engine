Other conventions\+:
\begin{DoxyItemize}
\item All defined variables are ALWAYS initialized
\item Four spaces are used, instead of TABS
\item Trailing spaces are always avoided
\item Control flow statements are followed {\bfseries{by a space}}\+:

\DoxyCodeLine{\textcolor{keywordflow}{switch}\ (value)}

\begin{DoxyItemize}
\item Directories are named using {\ttfamily snake\+\_\+case}\+: {\ttfamily resources/models}, {\ttfamily resources/fonts}
\item Files are named using {\ttfamily snake\+\_\+case}\+: {\ttfamily main\+\_\+title.\+png}, {\ttfamily cubicmap.\+png}, {\ttfamily sound.\+wav}
\end{DoxyItemize}

{\itshape NOTE\+: Spaces and special characters are always avoided in the files/dir naming!}\hypertarget{md__c_o_n_v_e_n_t_i_o_n_s_autotoc_md2}{}\doxysection{\texorpdfstring{Games/\+Examples Directories Organization Conventions}{Games/\+Examples Directories Organization Conventions}}\label{md__c_o_n_v_e_n_t_i_o_n_s_autotoc_md2}

\begin{DoxyItemize}
\item Resource files are organized by context and usage in the game. Loading requirements for data are also considered (grouping data when required).
\item Descriptive names are used for the files, just reading the name of the file it should be possible to know what is that file and where fits in the game.
\end{DoxyItemize}
